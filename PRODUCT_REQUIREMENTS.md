# vibecoding App 产品需求文档 (PRD)

> **重要提示**：本文档定义了产品的核心定位和功能，未经产品负责人明确许可，任何 AI 或开发人员**绝对不可删减、篡改**以下核心内容。

## 1. 核心定位（不可变更）

### 1.1 一句话定位
面向有学习记录习惯的年轻人（学生、考研党、日记党、职场新人）的学习记录工具，支持每日进展记录、致未来的信、AI 鼓励与学习目标/路径规划。

### 1.2 目标用户
核心为有学习记录习惯的年轻人，包含学生、考研党、日记党、职场新人。

## 2. 核心功能（必须保留，绝对不可删减/篡改）

| 功能序号 | 功能描述 | 核心要求 |
| :--- | :--- | :--- |
| **1** | **每日进展记录与历史查看** | 支持按「日期 + 标题 + 正文」框架记录当日进展；支持通过选择日期检索历史记录 |
| **2** | **致未来的信** | 提供写给 3 年 / 5 年 / 10 年后自己的写信功能 |
| **3** | **AI 近三日记录分析与鼓励** | 基于用户近 3 日记录，通过 AI 大模型生成个性化鼓励内容 |
| **4** | **AI 学习目标与路径生成** | 基于用户记录，AI 大模型生成清晰的学习目标 + 最快达成的学习路径建议 |

## 3. 技术栈（不可变更）

### 3.1 前端架构
*   **核心框架**：React 19（支持并发渲染）
*   **编程语言**：TypeScript（类型安全）
*   **构建工具**：Vite 6（快速热更新 / 打包）
*   **路由管理**：React Router 7（多页面跳转）
*   **状态管理**：React Context API（UserContext 管理登录 / 全局数据）
*   **样式处理**：Tailwind CSS（响应式，适配手机 / 平板）

### 3.2 后端架构
*   **运行环境**：Node.js
*   **服务端框架**：Express
*   **开发辅助**：tsx（直接运行 TypeScript）
*   **安全性**：bcrypt（密码加密）
*   **文件上传**：multer（头像等静态资源）

### 3.3 数据持久化
*   **数据库**：Supabase（PostgreSQL，实时同步）
*   **客户端**：@supabase/supabase-js（前后端统一操作）
*   **表结构**：schema.sql（定义用户、学习会话、目标等核心表）

### 3.4 外部集成
*   **AI 功能**：DeepSeek API（DeepSeek-Chat 模型）

### 3.5 部署与运维
*   **静态部署**：GitHub Pages（前端）+ GitHub Actions（自动构建 / 部署）
*   **环境管理**：dotenv（隔离敏感信息）

### 3.6 开发流程
*   **代码规范**：TypeScript 严格模式
*   **自动化**：GitHub Actions（自动打包、部署）

## 4. 产品形态（不可变更）

### 4.1 基础属性
非纯前端轻应用，需支持「用户名 + 密码」登录，具备云同步能力。

### 4.2 界面风格
核心为「极简 + 治愈」风格，无其他风格变体。

### 4.3 核心交互流程
1.  **打开 App**
2.  **登录**
3.  **写日记**（按日期 + 标题 + 正文框架）并保存
4.  **AI 大模型分析内容生成鼓励**
5.  **退出 App**
    *   *后续可扩展历史记录查看、写信、AI 目标生成等分支流程，但核心主流程不可改*。
